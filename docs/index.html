Introduction

Trio is an open source toolkit for building static webistes with HTML, CSS and JavaScript. With almost "zero configuration" required you use YAML front matter, JavaScript and Sass/CSS to quickly prototype your designs, which makes Trio the perfect tool for those who love to design in the browser.

As such Trio is somewhat opinionated but if you love working with HTML, CSS and JavaScript then you just might love creating static websites with it.

This isn't just la di da!

About This Document

Throughout this document the use of the term content should be interpreted to mean both HTML and markdown.

Features

  • Through the use of convention over configuration, Trio requires almost zero configuration, which means you can start using Trio right out of the box.

  • Supports Sass (i.e. .scss, .sass, .map ) and vanila CSS. See Project Structure for more information.

  • Trio is extensible and should you need to you can easily add your own pre and post processing using Grunt, Gulp, NPM, etc. task-based workflows.

  • Permalinks (sometimes called "pretty links") are dynamically generated for all pages. See URLS for more information.

  • Prepends URLs found in src and href attributes with a URL base segment at time of release. See configuration and command line for more information.

  • Cache busts your files using MD5 hash-based file names at time of release.

  • Extensive support for blogging including tags, categories and numerous catalogs of blog-specific metadata that are exposed to JavaScript callbacks used to customize your blog pages.

  • Metadata generated at build time is saved to file "trio.manifest.json" in your project's root folder. Having the ability to actually see this data makes it much easier to reason about when coding the JavaScript callbacks you use to dynamically customize your web pages.

  • Simple and easy to reason about project structure.

  • Front Matter parses as HTML comments and doesn't break markup and markdown formatting.

  • Comprehensive command line support is provided:

    • Create new project
    • Build project for development
    • Launch the browser and watch for changes
    • Build project for release
    • Help

Installing Trio

To install Trio, run the following from the command line:

npm i -g @4awpawz/trio

Project Structure

root/

The project's root folder.

root/public/

The folder where the generated website is saved to and is recreated whenever Trio builds the project's source folder.

root/source/

The Trio project's development folder. All development is confined to this folder.

root/source/callbacks/

The folder where JavaScript callback modules are kept. These are the JavaScript modules you declare in both page fragments and includes.

root/source/css/

The folder where CSS files are kept.

root/source/data/

The folder where .json data files are kept which are exposed via the dataCatalog property in the metadata that Trio generates and their contents are exposed as "file name": [file content].

For example, the content of the file /data/personalization.json would be exposed as:

"dataCatalog": {
    "personalization": {
        "name": "John Doe",
        "twitter": "@johndoe"
    },
    "someotherfile" {...}
}

root/source/fragments/

The folder where .md and .html page fragments are kept.

root/source/fragments/blog

The folder where .md and .html blog specific page fragments are kept.

root/source/fragments/blog/articles

The folder where .md and .html blog article specific page fragments are kept.

root/source/includes/

The folder where .md and .html include files are kept.

root/source/media/

The folder where media (.jpg, .gif, .pdf, .etc) are kept.

root/source/sass/

The folder where the main and imported .scss files are kept.

root/source/scripts/

The folder where run-time JavaScript files are kept.

root/source/templates/

The folder where .html page templates are kept.

root/trio.json

The project's configuration file.

root/trio.manifest.json

Contains the extensive collections of metadata that is generated every time Trio builds the project.
